# 项目结构文档

本项目是一个前后端分离的 **机房设备可视化管理系统**。

*   **后端 (`machine-backend`)**: 基于`SpringBoot`，提供`RESTful API`和业务逻辑。
*   **前端 (`machine-front`)**: 基于`Vue3`，实现数据展示和用户交互。

---

## **`machine-front` (前端项目)**

前端采用`Vue 3`全家桶和`Vite`构建，实现了数据展示、用户交互和状态管理。

```
machine-front/
├── .gitignore         # Git忽略文件配置。
├── README.md          # 项目说明文件。
├── index.html         # 单页面应用的HTML主入口，Vue应用会挂载到这个页面上。
├── package.json       # Node.js项目配置文件。定义了项目依赖、脚本命令（如 dev, build）。
├── package-lock.json  # 精确锁定项目依赖的版本，确保团队成员安装的依赖版本一致。
├── vite.config.js     # Vite构建工具的配置文件。
├── public/            # 存放不会被Vite处理的静态资源，会被直接复制到构建输出目录。
└── src/               # 项目源代码主目录。
    ├── App.vue        # Vue应用的根组件，所有页面视图都在此组件内渲染。
    ├── main.js        # Vue应用的入口文件，负责初始化Vue实例、路由、状态管理等。
    ├── style.css      # 全局CSS样式文件。
    ├── assets/        # 存放会被Vite处理的静态资源（如图片、样式文件）。
    ├── components/    # 存放可复用的Vue组件。
    │   ├── ChangePasswordModal.vue # 修改密码模态框组件。
    │   ├── LoginModal.vue # 登录模态框组件。
    │   └── NavBar.vue     # 顶部导航栏组件。
    ├── router/        # 存放Vue Router的路由配置。
    │   └── index.js   # 路由配置文件，定义了所有页面的URL路径和对应组件。
    ├── store/         # 存放Pinia的状态管理模块（Store）。
    │   ├── auth.js    # 负责处理用户认证状态（如token,用户信息）的Store。
    │   └── content.js # 负责处理系统内容（如轮播图数据）的Store。
    └── views/         # 存放页面级视图组件。
        ├── Home.vue     # 系统主页/仪表盘视图。
        ├── cabinet/
        │   └── CabinetList.vue # 机柜管理页面视图。
        ├── device/
        │   └── DeviceList.vue  # 设备管理页面视图。
        ├── interface/
        │   └── InterfaceList.vue # 接口管理页面视图。
        └── room/
            └── RoomList.vue    # 机房管理页面视图。
```
